Part B: Basic Networking Concepts
* IP Address: An Internet Protocol (IP) address is a unique string of numbers separated by periods (IPv4) or colons (IPv6) that identifies a device on the internet or a local network. It is effectively the digital "mailing address" for a device.
* MAC Address: A Media Access Control (MAC) address is a unique, physical identifier assigned to a network interface controller (NIC) by the manufacturer. Unlike an IP address, which can change, a MAC address is permanent to the hardware.
* Default Gateway: This is the node (usually your router) that serves as an access point to other networks. If your device wants to send data to an IP address outside your local network (like a website), it sends it to the default gateway first.
* DNS: The Domain Name System (DNS) is the phonebook of the internet. It translates human-readable web addresses (like google.com) into machine-readable IP addresses (like 142.250.190.46) so browsers can load internet resources.
* Public IP vs Private IP: * Public IP: Assigned by your ISP, this is unique across the entire global internet and allows you to communicate with the outside world.
   * Private IP: Assigned by your local router, this is used only within your internal home or business network to allow local devices to talk to each other securely without exposing them directly to the internet.

Part A: Network Devices Research
Device
	Purpose
	How it Works
	Real-World Usage
	1. Router
	Connects multiple networks together and routes data between them.
	It inspects the destination IP address of incoming data packets and determines the most efficient path to forward them to their destination.
	Connecting your home Local Area Network (LAN) to the wider internet (WAN).
	2. Switch
	Connects multiple devices within a single local network.
	It uses MAC addresses to forward data packets strictly to the specific device that requested them, rather than broadcasting to everyone.
	Used in corporate offices to wire dozens of desktop computers and printers together efficiently.
	3. Hub
	A legacy device used to connect multiple devices in a local network.
	It is a "dumb" device; when it receives data, it simply broadcasts copies of that data to all connected ports, causing network congestion.
	Mostly obsolete today, but historically used for very small, cheap network setups.
	4. Access Point
	Provides wireless (Wi-Fi) connectivity to a wired network.
	It connects directly to a router or switch via an Ethernet cable and projects a Wi-Fi signal to a designated area.
	Used in large buildings, universities, or airports to extend Wi-Fi coverage across a wide physical area.
	5. Firewall
	Protects the network by filtering incoming and outgoing traffic.
	It follows a set of pre-configured security rules to block malicious traffic and allow safe traffic, acting as a security guard between networks.
	Deployed at the edge of corporate networks to prevent hackers from accessing internal company databases.
	6. Modem
	Translates data between your home network and the ISP's infrastructure.
	It modulates digital signals from your router into analog signals for cables/phone lines, and demodulates incoming analog signals back into digital data.
	The box your Internet Service Provider (ISP) gives you to connect your house to their fiber, cable, or DSL lines.
	

Part B: IP Address Classification
1. 192.168.1.10  -> Private. Reason: This falls strictly into the Class C private IP range (192.168.0.0 – 192.168.255.255), reserved for local networks.
2. 10.0.0.5  -> Private. Reason: This falls into the Class A private IP range (10.0.0.0 – 10.255.255.255), heavily used in corporate environments.
3. 172.16.5.20  -> Private. Reason: This is within the Class B private IP range (172.16.0.0 – 172.31.255.255).
4. 8.8.8.8  -> Public. Reason: This is a publicly routable IP address on the internet (specifically, Google's public DNS server). It does not fall into any of the reserved private ranges.
5. 1.1.1.1  -> Public. Reason: Similar to the above, this is a globally unique public IP (Cloudflare's DNS) and is reachable directly via the internet.
6. 192.168.100.1  -> Private. Reason: This is in the Class C private block. Often used as the default gateway IP for consumer routers.

Part D: Network Communication Flow


Flow for opening www.google.com:
[Your Device] --> [Router] --> [DNS Server] --> [Google Server] --> [Response Back to Device]


Step-by-Step Explanation:
1. Your Device: You type www.google.com into your browser. Your device realizes it doesn't know the IP address for this domain, so it prepares a DNS query.
2. Router: Your device sends this query to your local router (the Default Gateway), which forwards the request out to the internet to reach your configured DNS provider.
3. DNS Server: The DNS server looks up www.google.com, finds its corresponding public IP address (e.g., 142.250.190.46), and sends this information back to your device.
4. Google Server: Now armed with the correct IP address, your device sends an HTTP/HTTPS request directly to Google's server asking for the webpage data.
5. Response Back to Device: Google's server processes the request and sends the website's HTML/CSS/JavaScript packets back through the internet, through your router, and finally to your screen.
