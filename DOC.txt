Part B: Basic Networking Concepts
* IP Address: An Internet Protocol (IP) address is a unique string of numbers separated by periods (IPv4) or colons (IPv6) that identifies a device on the internet or a local network. It is effectively the digital "mailing address" for a device.
* MAC Address: A Media Access Control (MAC) address is a unique, physical identifier assigned to a network interface controller (NIC) by the manufacturer. Unlike an IP address, which can change, a MAC address is permanent to the hardware.
* Default Gateway: This is the node (usually your router) that serves as an access point to other networks. If your device wants to send data to an IP address outside your local network (like a website), it sends it to the default gateway first.
* DNS: The Domain Name System (DNS) is the phonebook of the internet. It translates human-readable web addresses (like google.com) into machine-readable IP addresses (like 142.250.190.46) so browsers can load internet resources.
* Public IP vs Private IP: * Public IP: Assigned by your ISP, this is unique across the entire global internet and allows you to communicate with the outside world.
   * Private IP: Assigned by your local router, this is used only within your internal home or business network to allow local devices to talk to each other securely without exposing them directly to the internet.

Answers to Questions:
1. Was the ping successful?  Yes, the ping was successful. The command received replies from the destination server, indicating that the device can reach Google's servers.
2. How many hops were shown?  In the dummy tracert output provided above, there were 3 hops. The first hop was the local router, the second was the ISP's infrastructure, and the final hop was the Google server.
3. What is the purpose of traceroute?  Traceroute maps the exact journey a packet of data takes from your device to its destination. It records every router (hop) the data passes through and the time it takes between each, which is highly useful for diagnosing where a network delay or failure is occurring.
